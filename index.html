HTML

  • HTML is used to create the structure of a webpage.
  • HTML is made up of elements which work together to provide content and the basic framework for all pages on the web.
  • An element is a type of content on a webpage.
  • Elements that are nested inside other elements are called child elements.
  • At the begining part of HTML code, we have the following:
    • !DOCTYPE html declaration which tells the browser what type of document it should expect when it's opened.
    • Next, we have html opening tag, it represents the start of our html document and inside the html opening tag, there is an attribute of lang=en, which tells the language for the webpage..

      • Inside the html opening and closing tags, we have:

        • The head element, which contains information about the webpage.

          • Inside the head element, we can use the following metadata tags:
            • The title element, defines the title for a webpage.
            • The style element, defines the css styles associated with a website.
            • The base element, defines the base url for a webpage.
            • The link element, connects an external resource to the HTML document.
            • The meta element, defines metadata such as the character set, description, keywords author and viewpoint.
            • The script element, embeds client-side scripts in the HTML document.

        • The body element, which represents the visible content shown to the user.

          • Inside the body element, we can use the following main child elements:
            • The header element, appears at the top of the page and usually contains a navigation menu and/or introductory content.
            • The main element, contains the main code of the webpage.
              • Inside main element, we can have section element, a semantic element that defines a section in a document that contains semantic content.
            • The footer element, appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation.

          • For each of the above main child elements, we can have the following additional elements if necessary:
            • The heading element (h1, h2, h3, h4, h5 and h6), represents level of heading a given text blockrepresents.
            • The p element, represents a paragraph.
            • The ul (unordered list), ol (ordered list) and li(list items), represent bulleted or numbered list.
            • The img element, contains info about the images that displayed on the webpage.
              • This img element can contain various attributes:
                • The src attribute is the most important because it defines the location of the image.
                • The alt attribute, contains a text string that describes the appearance and functionality of the image.
            • The br element, creates a line of empty space or a break between 2 blocks.
            • The a (anchor) element, creates links to the same webpage or other webpages.
              • Inside the a element, we can have href attribute which points to the url for the link.
  • The following element types are commonly used to add content to pages:
    • text elements
    • Image elements
    • Header elements
    • List elements
  • The following element types are used to devide and describe a page:
    • Meta-related elements
    • Content division elements
    • Header elements
    • Semantic elements

CSS

  • CSS (Cascading Style Sheet), a programming language that determines the style of a webpage in order to make a website visually appealing.
  • There are 3 ways to style a webpage using CSS:
    • Inline CSS which is used when a developer wants to style an element directly in the HTML file.
    • Internal style sheet, contains CSS rules for the webpage in the head section of the HTML file.
    • External CSS style sheet, is a preferred method in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • A CSS rule is made of selectors and declaration.
    • Selector defines the element (eg: header, footer or h1) or attributes (eg: class) to which the rules or declarations will apply to.
      • When adding a rule to an attribute selector like a class, preceed it with a period (.) in the CSS file.
    • Declarations contain 2 important components: CSS property (eg: color) and the value of the property (eg: blue)
    • To apply a rule to multiple selectors at once, you can separate them by a coma.
  • Some of the common properties are:
    • A margin, which indicates how much space we want around the outside of an element.
    • A padding, which indicates how much we want around the content inside an element.
    • Display.
    • Height.
    • width.
    • Border.
    • Text-align.
    • Font-size.
    • Box-shadow.
    • Color.
    • Etc...
  • When using an external css style sheet, we link the the file using a link element placed in the HTML's head element.
    • This link element has 2 attributes: rel and href.
      • The rel attribute specifies the relationship between the current document and the linked document resource.
      • The href attribute specifies the url of the external source.
    • The link element is always placed in HTML's head element and it comes after meta element and before title element.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: adds all changed files in staging area.
  • git commit -m commit-message: commit changes to the current branch.
  • git pull origin main: pulls remote main branch content to the current local branch.
  • git push origin branch-name: pushes modifications in the mentioned branch name to GitHub.
